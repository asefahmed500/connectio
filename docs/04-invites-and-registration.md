# 04 — Invites & Registration

**Status:** Draft
**Models:** `Invite`, `User`, `Client`
**Routes:** `/invite/[slug]` (public) · `POST /api/auth/register` (Server Action under `/invite/[slug]`)

There is no public sign-up. The only way to become a `User` is through an invite link generated by a Super Admin.

## Invite lifecycle

```
       ┌─────────┐  admin generates    ┌──────────┐
admin  │  (none) │ ──────────────────▶ │   OPEN   │
       └─────────┘                     └────┬─────┘
                                            │ client clicks link,
                                            │ registers
                                            ▼
                                       ┌──────────┐
                                       │ CONSUMED │ ────▶ Client + User rows created
                                       └──────────┘
                                            ▲
                                            │ admin cancels
              admin revokes ──────▶  ┌──────────┐
                                     │ REVOKED  │
                                     └──────────┘

              time passes (expiresAt) ─▶  ┌──────────┐
                                         │ EXPIRED  │  (set lazily by reads;
                                         └──────────┘   no cron required for correctness)
```

`InviteStatus` transitions are one-way except `OPEN → EXPIRED ← OPEN` (lazy). No `CONSUMED → OPEN`.

## Slug generation

`Invite.slug` becomes `Client.uniqueSlug` on registration. Constraints:

- Lowercase ASCII letters, digits, hyphens.
- 3–32 chars.
- Globally unique (the `@unique` constraint catches collisions).
- Reserved-word blocklist: `admin`, `team`, `api`, `auth`, `invite`, `dashboard`, `login`, `register`, `settings`, `www`.

Generation strategy:

```ts
// lib/dal/invites.ts
import 'server-only'
import { randomSlug } from '@/lib/slug'

async function proposeSlug(input: { contactName: string; companyName: string }): Promise<string> {
  // Prefer a cleaned-up version of contactName; fall back to company; then random.
  const candidates = [
    slugify(input.contactName),
    slugify(input.companyName),
    `${slugify(input.contactName)}-${randomSuffix(4)}`,
    randomSlug(8),
  ]
  for (const c of candidates) {
    if (await isSlugAvailable(c)) return c
  }
  throw new Error('Unreachable: randomSlug(8) should always be available')
}

function slugify(s: string): string {
  return s.toLowerCase().trim()
    .replace(/[^a-z0-9]+/g, '-')
    .replace(/^-+|-+$/g, '')
    .slice(0, 32)
}
```

## Generate invite (admin)

Server Action `createInviteAction`. Flow:

1. `requireRole('SUPER_ADMIN')`.
2. Validate input with Zod.
3. Generate slug.
4. Insert `Invite` row with `expiresAt = now + 7 days`.
5. Fire email to `invite.email` with link `${APP_URL}/invite/${slug}` via `after()`.
6. Return `{ slug, inviteLink }` for the admin UI.

```ts
// app/(admin)/invites/actions.ts
'use server'

import { z } from 'zod'
import { after } from 'next/server'
import { requireRole } from '@/lib/dal/session'
import { createInvite } from '@/lib/dal/invites'
import { sendInviteEmail } from '@/lib/email'

const Schema = z.object({
  email: z.email(),
  companyName: z.string().min(1).max(120),
  contactName: z.string().min(1).max(120),
})

export async function createInviteAction(prevState: unknown, formData: FormData) {
  const claims = await requireRole('SUPER_ADMIN')
  const parsed = Schema.safeParse({
    email: formData.get('email'),
    companyName: formData.get('companyName'),
    contactName: formData.get('contactName'),
  })
  if (!parsed.success) return { error: 'Invalid input' }

  const invite = await createInvite({ ...parsed.data, createdBy: claims.sub })

  after(async () => {
    await sendInviteEmail({
      to: invite.email,
      inviteLink: `${process.env.NEXT_PUBLIC_APP_URL}/invite/${invite.slug}`,
      companyName: invite.companyName,
    })
  })

  return { success: true, slug: invite.slug }
}
```

## Public invite page (`/invite/[slug]`)

This is the only registration entry point. Server Component:

```tsx
// app/invite/[slug]/page.tsx
import { notFound } from 'next/navigation'
import { getInviteForRegistration } from '@/lib/dal/invites'
import { RegisterForm } from './register-form'

export default async function InvitePage({ params }: { params: Promise<{ slug: string }> }) {
  const { slug } = await params
  const invite = await getInviteForRegistration(slug)  // returns null if not OPEN
  if (!invite) notFound()
  return <RegisterForm invite={invite} />
}
```

`getInviteForRegistration(slug)` does:
- Look up by slug.
- If `status !== OPEN`, return null.
- If `expiresAt < now`, set `status = EXPIRED` (lazy) and return null.
- Return a DTO with the fields the form needs (`email`, `companyName`, `contactName`, `slug`).

The DTO is intentionally minimal — no admin user ID, no internal fields.

## Registration flow

The `RegisterForm` is a Client Component wired to a Server Action via `useActionState`. The action validates everything server-side again — never trust the hidden form fields.

```ts
// app/invite/[slug]/actions.ts
'use server'

import { z } from 'zod'
import { cookies } from 'next/headers'
import { redirect } from 'next/navigation'
import { hashPassword } from '@/lib/auth/password'
import { signAccessToken, generateRefreshToken } from '@/lib/auth/tokens'
import { hashRefreshToken } from '@/lib/auth/session'
import { prisma } from '@/lib/db'
import { writeAudit } from '@/lib/audit'
import { consumeInvite } from '@/lib/dal/invites'

const Schema = z.object({
  slug: z.string().min(3).max(32).regex(/^[a-z0-9-]+$/),
  email: z.email(),
  password: z.string().min(12).regex(/[A-Za-z]/).regex(/\d/).regex(/[^A-Za-z0-9]/),
  name: z.string().min(1).max(120),
})

export async function registerAction(prevState: unknown, formData: FormData) {
  const parsed = Schema.safeParse({
    slug: formData.get('slug'),
    email: formData.get('email'),
    password: formData.get('password'),
    name: formData.get('name'),
  })
  if (!parsed.success) {
    return { error: 'Invalid input', fields: parsed.error.flatten().fieldErrors }
  }

  // Single transaction: validate invite, create user+client, consume invite.
  const result = await prisma.$transaction(async (tx) => {
    const invite = await tx.invite.findUnique({ where: { slug: parsed.data.slug } })
    if (!invite || invite.status !== 'OPEN' || invite.expiresAt < new Date()) {
      throw new RegistrationError('INVITE_INVALID', 'This invite link is no longer valid.')
    }
    if (invite.email.toLowerCase() !== parsed.data.email.toLowerCase()) {
      throw new RegistrationError('EMAIL_MISMATCH', 'Email does not match the invite.')
    }
    const existing = await tx.user.findUnique({ where: { email: parsed.data.email } })
    if (existing) {
      throw new RegistrationError('EMAIL_TAKEN', 'An account with this email already exists.')
    }

    const user = await tx.user.create({
      data: {
        email: parsed.data.email,
        name: parsed.data.name,
        passwordHash: await hashPassword(parsed.data.password),
        role: 'CLIENT',
        client: {
          create: {
            companyName: invite.companyName,
            contactName: invite.contactName,
            uniqueSlug: invite.slug,
            invite: { connect: { id: invite.id } },
          },
        },
      },
      include: { client: true },
    })

    await tx.invite.update({
      where: { id: invite.id },
      data: { status: 'CONSUMED', consumedBy: user.id },
    })

    return user
  })

  // Issue tokens outside the tx (cookie writes shouldn't block commit).
  const access = await signAccessToken({ sub: result.id, role: 'CLIENT', clientId: result.client!.id })
  const refresh = generateRefreshToken()
  await prisma.session.create({
    data: {
      userId: result.id,
      refreshTokenHash: await hashRefreshToken(refresh),
      expiresAt: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000),
    },
  })
  const cs = await cookies()
  cs.set('access_token', access, { httpOnly: true, secure: true, sameSite: 'lax', path: '/', maxAge: 60 * 60 * 24 })
  cs.set('refresh_token', refresh, { httpOnly: true, secure: true, sameSite: 'lax', path: '/', maxAge: 60 * 60 * 24 * 7 })

  await writeAudit('USER_REGISTERED', result.id, 'User', result.id)

  redirect(`/dashboard/visitor/${result.client!.uniqueSlug}`)
}
```

### Why a single transaction?

If we created the `User` then crashed before marking `Invite.status = CONSUMED`, the client could re-use the link. Worse: if we created the User but failed on `Client`, we'd have an orphaned user with no client profile. Wrapping all three writes in a Prisma `$transaction` makes it atomic.

### Why doesn't this use `after()` for the session creation?

Because the user needs to land on their dashboard already authenticated. The session write is on the response path, not a side effect.

## Revoke invite

`revokeInviteAction(slug)` — `requireRole('SUPER_ADMIN')`, sets `status = REVOKED` only if currently `OPEN`. Idempotent.

## Resend invite email

`resendInviteAction(slug)` — same guard, only if `OPEN` and `expiresAt > now`. Rate-limited (max 3 resends per invite per hour). Re-sends the same link; we don't rotate the slug.

## Expiry handling

No dedicated cron. Lazy on read:
- `getInviteForRegistration(slug)` flips `OPEN → EXPIRED` if past `expiresAt`.
- A weekly cleanup job (Vercel Cron, `13-deployment.md`) deletes CONSUMED/REVOKED invites older than 90 days for tidiness.

## Security notes

- **Slugs are not secret.** Anyone with the link can attempt registration. The invite's `email` field is the gate: a stranger who guesses a slug cannot register because their email won't match. (For higher security in future, add a random unguessable token to the URL: `/invite/<slug>/<token>`. Tracked as a v2 hardening item.)
- **Email-match check happens server-side in the transaction.** Hidden form fields and pre-filled values are convenience, not validation.
- **Registration rate-limited** per IP (5/hour) and per slug (3/hour).
