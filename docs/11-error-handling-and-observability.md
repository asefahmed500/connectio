# 11 — Error Handling & Observability

**Status:** Draft

Errors are split into four categories, each with its own handling rule:

| Category | Example | Handling |
|----------|---------|----------|
| **User input errors** | Bad email, missing field | Return `{ fields }` to the form; 400 from route handlers |
| **Auth errors** | No session, wrong role | `unauthorized()` / `forbidden()` → dedicated pages; 401/403 from route handlers |
| **Domain errors** | Invalid state transition, slug taken | Throw typed error; render message inline or 422 |
| **System errors** | DB down, S3 timeout | Caught by `error.tsx` / `global-error.tsx`; 500; log + trace |

## Typed errors

```ts
// lib/errors.ts
export class AppError extends Error {
  constructor(
    message: string,
    public code: string,
    public httpStatus: number = 500,
    public fields?: Record<string, string[]>,
  ) {
    super(message)
    this.name = this.constructor.name
  }
}

export class ValidationError  extends AppError { constructor(fields) { super('Validation failed', 'VALIDATION', 400, fields) } }
export class UnauthorizedError extends AppError { constructor() { super('Unauthorized', 'UNAUTHORIZED', 401) } }
export class ForbiddenError    extends AppError { constructor() { super('Forbidden', 'FORBIDDEN', 403) } }
export class NotFoundError     extends AppError { constructor(what: string) { super(`${what} not found`, 'NOT_FOUND', 404) } }
export class ConflictError     extends AppError { constructor(what: string) { super(what, 'CONFLICT', 409) } }
export class TransitionError   extends AppError { constructor(from: string, to: string) { super(`Invalid transition ${from} → ${to}`, 'INVALID_TRANSITION', 422) } }
export class RateLimitError    extends AppError { constructor(retryAfter: number) { super('Too many requests', 'RATE_LIMIT', 429) } }
```

## Where errors are caught

```
Server Action
   │  try { DAL call }
   │  catch (err):
   │    - ValidationError  → return { fields }
   │    - Auth errors      → re-throw (handled upstream by Next)
   │    - Domain errors    → return { error: err.message }
   │    - System errors    → log; return generic { error }
   │
Route Handler
   │  try { handler }
   │  catch (err):
   │    - AppError         → NextResponse.json with appropriate status
   │    - Prisma errors    → map to 4xx (e.g. P2002 → ConflictError)
   │    - everything else  → 500 with requestId
   │
Server Component
   │  Don't catch system errors — let error.tsx render.
   │  Catch only if you have a meaningful fallback (e.g. empty list).
```

## `error.tsx` boundaries

```tsx
// app/error.tsx
'use client'

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  return (
    <div className="grid place-items-center min-h-[60vh]">
      <div className="text-center">
        <h2>Something went wrong</h2>
        <p className="text-muted-foreground text-sm">
          Reference: {error.digest ?? 'no-ref'}
        </p>
        <button onClick={reset}>Try again</button>
      </div>
    </div>
  )
}
```

`error.digest` is a stable hash generated by Next for the error; we log it server-side and surface it in the UI so users can quote it in support requests.

`app/global-error.tsx` wraps `<html><body>` itself (it replaces the root layout on render):

```tsx
// app/global-error.tsx
'use client'

export default function GlobalError({ error, reset }) {
  return (
    <html lang="en">
      <body>
        <h2>Application error</h2>
        <p>Reference: {error.digest}</p>
        <button onClick={reset}>Reload</button>
      </body>
    </html>
  )
}
```

## Server Action contract

Server Actions can return anything JSON-serializable. Standardize on:

```ts
type ActionState<TSuccess = unknown> =
  | { success: true; data: TSuccess }
  | { success: false; error: string; fields?: Record<string, string[]> }
```

Used with `useActionState`:

```tsx
'use client'
import { useActionState } from 'react'
import { someAction } from './actions'

export function MyForm() {
  const [state, action, pending] = useActionState(someAction, { success: false, error: '' })
  // render based on state
}
```

## Navigation errors in try/catch

Per `next-best-practices/error-handling.md`: `redirect()`, `notFound()`, `unauthorized()`, `forbidden()` throw special errors that Next handles internally. **Do not catch them in your action's try/catch** — or use `unstable_rethrow`:

```ts
'use server'
import { unstable_rethrow } from 'next/navigation'

export async function action() {
  try {
    // ... work
    redirect('/somewhere')
  } catch (err) {
    unstable_rethrow(err)        // re-throws NEXT_REDIRECT, NOT_FOUND, etc.
    return { success: false, error: 'Something went wrong' }
  }
}
```

## Logging

Use a thin wrapper around `console` that:

1. Renders JSON in production, pretty in dev.
2. Sanitizes known-sensitive keys (`password`, `token`, `cookie`, `authorization`).
3. Adds `requestId` from headers (set by `proxy.ts`).
4. Adds `userId` from session when available.

```ts
// lib/log.ts
import 'server-only'

type Level = 'debug' | 'info' | 'warn' | 'error'

export const log = {
  debug: (msg: string, ctx?: object) => write('debug', msg, ctx),
  info:  (msg: string, ctx?: object) => write('info',  msg, ctx),
  warn:  (msg: string, ctx?: object) => write('warn',  msg, ctx),
  error: (msg: string, ctx?: object) => write('error', msg, ctx),
}

function write(level: Level, msg: string, ctx?: object) {
  const line = JSON.stringify({ level, msg, ts: new Date().toISOString(), ...sanitize(ctx) })
  if (level === 'error') console.error(line)
  else if (level === 'warn') console.warn(line)
  else console.log(line)
}

const SENSITIVE = /^(password|token|secret|cookie|authorization|refreshToken)$/i
function sanitize(ctx?: object): object | undefined {
  if (!ctx) return undefined
  return Object.fromEntries(
    Object.entries(ctx).map(([k, v]) => [k, SENSITIVE.test(k) ? '[redacted]' : v])
  )
}
```

In production, Vercel captures stdout/stderr into its log drain. Pipe to Logflare / Datadog / similar.

## Audit log (separate from app logs)

Application logs are for ops. The audit log is for accountability — it lives in the database, is queryable from the admin UI, and is **append-only**.

```ts
// lib/audit.ts
import 'server-only'
import { prisma } from '@/lib/db'

export async function writeAudit(
  action: string,
  userId: string | null,
  resource: string,
  resourceId: string,
  changes?: { before?: unknown; after?: unknown },
) {
  await prisma.auditLog.create({
    data: {
      userId,
      action,
      resource,
      resourceId,
      changes: changes ?? undefined,
      ip: await readIp(),
      userAgent: await readUserAgent(),
    },
  })
}
```

For writes that should be non-blocking, wrap in `after()`:

```ts
import { after } from 'next/server'

after(async () => {
  await writeAudit('SUBMISSION_APPROVED', claims.sub, 'Submission', id)
})
```

## Open Telemetry

Next 16 has built-in OTel instrumentation. Hook up the OTLP exporter in `instrumentation.ts`:

```ts
// instrumentation.ts
export async function register() {
  if (process.env.NEXT_RUNTIME === 'nodejs' && process.env.OTEL_EXPORTER_OTLP_ENDPOINT) {
    const { registerOTel } = await import('@vercel/otel/register')
    registerOTel({
      serviceName: 'connectio',
      traceExporter: { url: process.env.OTEL_EXPORTER_OTLP_ENDPOINT },
    })
  }
}
```

Spans cover route handlers, server actions, DB calls (Prisma), and HTTP fetches.

## Health check

`/api/health` returns:

```json
{ "ok": true, "time": "2026-06-14T10:00:00Z", "db": "up", "version": "1.0.0" }
```

No auth. Cacheable for 10s. Used by Vercel's deploy checks and external uptime monitors.

If the DB is unreachable, returns 503 with `{ ok: false, db: 'down' }`.
